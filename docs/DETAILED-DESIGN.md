## 1. 인증 시스템 상세 설계

### 1.1 가입 플로우

#### 1.1.1 소셜 로그인 가입 (Google, GitHub, Kakao)

**플로우 다이어그램:**

```
[랜딩 페이지]
    │
    ▼
[로그인/회원가입 페이지]
    │
    ├── "Google로 시작하기" 클릭
    │       │
    │       ▼
    │   [Google OAuth 동의 화면]
    │       │ (이메일, 프로필 사진, 이름 수집)
    │       ▼
    │   [OAuth Callback 처리]
    │       │
    │       ├── 기존 계정 존재 → 로그인 처리 → [대시보드]
    │       │
    │       └── 신규 사용자 → [온보딩 화면]
    │               │
    │               ├── Step 1: 약관 동의 (필수)
    │               │   ├── ☑ 서비스 이용약관 (필수)
    │               │   ├── ☑ 개인정보 처리방침 (필수)
    │               │   └── ☐ 마케팅 수신 동의 (선택)
    │               │
    │               ├── Step 2: 추가 프로필 (선택)
    │               │   ├── 직군 선택 (PM/PO, 개발자, 디자이너, 창업자, 기타)
    │               │   └── 경력 (주니어 0-2년, 미들 3-5년, 시니어 6년+)
    │               │
    │               └── [대시보드] (홈 화면으로 이동)
    │
    ├── "GitHub로 시작하기" 클릭
    │       └── (Google과 동일한 플로우, GitHub OAuth)
    │
    └── "카카오로 시작하기" 클릭
            └── (Google과 동일한 플로우, Kakao OAuth)
```

**각 소셜 프로바이더별 수집 데이터:**

| 프로바이더 | 수집 항목 | OAuth Scope |
|-----------|-----------|-------------|
| Google | 이메일, 이름, 프로필 사진 | `openid email profile` |
| GitHub | 이메일, 사용자명, 아바타 | `read:user user:email` |
| Kakao | 이메일, 닉네임, 프로필 이미지 | `profile_nickname profile_image account_email` |

**동일 이메일 소셜 계정 연결 정책:**

- 이미 가입된 이메일로 다른 소셜 로그인 시도 시: "이미 Google로 가입된 계정입니다. Google로 로그인한 후 계정 설정에서 GitHub 계정을 연결할 수 있습니다." 안내 메시지 표시
- 계정 설정 페이지에서 추가 소셜 계정 연결/해제 가능
- 최소 1개의 로그인 수단은 항상 유지 (모든 연동 해제 불가)

#### 1.1.2 이메일/패스워드 가입

**플로우 다이어그램:**

```
[로그인/회원가입 페이지]
    │
    └── "이메일로 가입" 클릭
            │
            ▼
        [가입 폼]
            │
            ├── 이메일 입력
            │   └── 실시간 중복 검사 (debounce 500ms)
            │
            ├── 비밀번호 입력
            │   └── 실시간 강도 표시 (약함/보통/강함)
            │
            ├── 비밀번호 확인 입력
            │   └── 실시간 일치 검증
            │
            ├── 이름 입력 (2-30자)
            │
            ├── 약관 동의 체크
            │   ├── ☑ 서비스 이용약관 (필수) [보기]
            │   ├── ☑ 개인정보 처리방침 (필수) [보기]
            │   └── ☐ 마케팅 수신 동의 (선택)
            │
            └── "가입하기" 클릭
                    │
                    ▼
                [이메일 인증 안내 화면]
                    │ "인증 메일을 발송했습니다"
                    │ (30분 내 인증 필요)
                    │
                    ▼
                [이메일 인증 링크 클릭]
                    │
                    ├── 토큰 유효 → [온보딩 화면] → [대시보드]
                    │
                    └── 토큰 만료/무효 → "인증 링크가 만료되었습니다"
                                          [인증 메일 재발송]
```

**비밀번호 정책:**

| 규칙 | 요구사항 | 표시 |
|------|---------|------|
| 최소 길이 | 8자 이상 | ☑/☐ |
| 최대 길이 | 72자 (bcrypt 제한) | - |
| 영문 포함 | 대문자 또는 소문자 1개 이상 | ☑/☐ |
| 숫자 포함 | 1개 이상 | ☑/☐ |
| 특수문자 포함 | 1개 이상 (`!@#$%^&*` 등) | ☑/☐ |
| 비밀번호 강도 | 위 조건 충족 수에 따라 약함/보통/강함 표시 | 프로그레스 바 |

**비밀번호 강도 시각화:**

```
약함 (2개 이하 충족)  ████░░░░░░ 빨간색
보통 (3개 충족)       ████████░░ 노란색
강함 (4개 모두 충족)  ██████████ 초록색
```

**이메일 인증 프로세스:**

```
1. 가입 시 crypto.randomUUID()로 인증 토큰 생성
2. 토큰을 EmailVerification 테이블에 저장 (TTL: 30분)
3. 인증 이메일 발송 (Resend API 사용)
   - 제목: "[PRD 생성기] 이메일 인증을 완료해주세요"
   - 본문: 인증 링크 (https://app.prd-gen.kr/verify-email?token=xxx)
4. 인증 전 상태: 로그인 가능하나, PRD 생성 3건 제한
5. 인증 완료: 모든 기능 사용 가능
6. 인증 미완료 사용자: 로그인 시 "이메일 인증 필요" 배너 상시 표시
```

**약관 동의 체크 구성:**

```
┌─────────────────────────────────────────────────┐
│ 약관 동의                                        │
│                                                  │
│ ☐ 전체 동의                                      │
│ ──────────────────────────────────────────────── │
│ ☑ [필수] 서비스 이용약관에 동의합니다     [보기 >] │
│ ☑ [필수] 개인정보 처리방침에 동의합니다   [보기 >] │
│ ☐ [선택] 마케팅 정보 수신에 동의합니다   [보기 >] │
└─────────────────────────────────────────────────┘
```

- "전체 동의" 체크 시 하위 전체 체크/해제
- 필수 항목 미동의 시 가입 버튼 비활성화
- 약관 내용은 모달 또는 새 탭으로 표시
- 동의 이력을 DB에 기록 (동의 일시, 약관 버전)

### 1.2 로그인 플로우

#### 1.2.1 소셜 로그인

```
[로그인 페이지]
    │
    ├── "Google로 로그인" 클릭
    │       │
    │       ▼
    │   [Google OAuth 화면]
    │       │
    │       ▼
    │   [서버 콜백 처리]
    │       │
    │       ├── 계정 존재 + 활성 → [대시보드]
    │       ├── 계정 존재 + 정지 → "계정이 정지되었습니다" 에러 메시지
    │       └── 계정 미존재 → [가입 플로우]로 전환
    │
    ├── "GitHub로 로그인"  → (동일 플로우)
    └── "카카오로 로그인"  → (동일 플로우)
```

#### 1.2.2 이메일/패스워드 로그인

```
[로그인 페이지]
    │
    └── 이메일 + 패스워드 입력 → "로그인" 클릭
            │
            ├── 성공 → [대시보드]
            │   └── "로그인 상태 유지" 체크 시
            │       → 세션 만료: 30일 (기본: 24시간)
            │
            ├── 실패: 이메일 미존재
            │   → "등록되지 않은 이메일입니다" (가입 링크 포함)
            │
            ├── 실패: 패스워드 불일치
            │   → "비밀번호가 일치하지 않습니다"
            │   → 5회 연속 실패 시: 15분 잠금 + "비밀번호 찾기" 안내
            │
            └── 실패: 계정 정지
                → "계정이 정지되었습니다. 문의: support@prd-gen.kr"
```

**로그인 상태 유지 정책:**

| 옵션 | 세션 유효 기간 | 쿠키 설정 |
|------|-------------|-----------|
| 체크 해제 (기본) | 24시간 | `maxAge: 86400` |
| 체크 (로그인 상태 유지) | 30일 | `maxAge: 2592000` |

**로그인 시도 제한 (Rate Limiting):**

| 조건 | 제한 | 해제 조건 |
|------|------|----------|
| 동일 이메일 | 5회/15분 | 15분 경과 또는 비밀번호 찾기 완료 |
| 동일 IP | 20회/15분 | 15분 경과 |
| 전체 시스템 | 1,000회/분 | 자동 해제 |

#### 1.2.3 비밀번호 찾기/재설정

```
[로그인 페이지]
    │
    └── "비밀번호를 잊으셨나요?" 클릭
            │
            ▼
        [비밀번호 재설정 요청]
            │
            ├── 이메일 입력 → "재설정 링크 발송" 클릭
            │
            ▼
        [이메일 발송 완료 안내]
            │ "비밀번호 재설정 링크를 발송했습니다"
            │ (보안: 미등록 이메일이어도 동일 메시지 표시)
            │
            ▼
        [이메일 내 링크 클릭] (유효 시간: 1시간)
            │
            ├── 토큰 유효
            │       │
            │       ▼
            │   [새 비밀번호 설정]
            │       ├── 새 비밀번호 입력 (정책 검증)
            │       ├── 비밀번호 확인
            │       └── "비밀번호 변경" 클릭
            │               │
            │               ▼
            │           [변경 완료]
            │           "비밀번호가 변경되었습니다"
            │           → 모든 기존 세션 무효화
            │           → [로그인 페이지]로 이동
            │
            └── 토큰 만료/무효
                → "링크가 만료되었습니다. 다시 요청해주세요."
                → [비밀번호 재설정 요청]으로 이동
```

**보안 고려사항:**

- 재설정 링크는 1회 사용 후 무효화
- 재설정 토큰은 SHA-256 해시하여 DB에 저장 (원문 저장 금지)
- 비밀번호 변경 시 모든 활성 세션 즉시 무효화
- 이전 3개 비밀번호 재사용 금지

### 1.3 인증 아키텍처

#### 1.3.1 NextAuth.js v5 (Auth.js) 기반 세션 관리

**선택: DB 세션 (Database Strategy)**

| 비교 항목 | JWT 전략 | DB 세션 전략 | **선택** |
|-----------|---------|------------|---------|
| 세션 저장 | 클라이언트 쿠키 | 서버 DB (sessions 테이블) | **DB** |
| 실시간 무효화 | 불가 (만료까지 유효) | 즉시 가능 (DB 삭제) | **DB** |
| 확장성 | 매 요청마다 DB 조회 불필요 | 매 요청마다 DB 조회 | JWT 유리 |
| 보안 | 토큰 탈취 시 만료까지 유효 | 토큰 탈취 시 서버에서 즉시 무효화 | **DB** |
| 사용자 차단 | 실시간 반영 불가 | 즉시 반영 | **DB** |
| 팀 권한 변경 | 재로그인 필요 | 즉시 반영 | **DB** |

**선택 근거:**
1. 관리자의 사용자 차단/정지를 즉시 반영해야 함
2. 팀 역할/권한 변경이 실시간으로 적용되어야 함
3. 비밀번호 변경 시 모든 세션을 즉시 무효화해야 함
4. Neon PostgreSQL의 커넥션 풀링으로 DB 조회 성능 확보 가능
5. Vercel Edge Runtime에서는 DB 세션의 지연이 미미 (같은 리전)

**Auth.js 설정 구조:**

```typescript
// src/lib/auth/config.ts
import NextAuth from "next-auth";
import { PrismaAdapter } from "@auth/prisma-adapter";
import Google from "next-auth/providers/google";
import GitHub from "next-auth/providers/github";
import Kakao from "next-auth/providers/kakao";
import Credentials from "next-auth/providers/credentials";

export const { handlers, auth, signIn, signOut } = NextAuth({
  adapter: PrismaAdapter(prisma),
  session: { strategy: "database", maxAge: 24 * 60 * 60 }, // 24시간
  providers: [
    Google({ clientId: env.GOOGLE_ID, clientSecret: env.GOOGLE_SECRET }),
    GitHub({ clientId: env.GITHUB_ID, clientSecret: env.GITHUB_SECRET }),
    Kakao({ clientId: env.KAKAO_ID, clientSecret: env.KAKAO_SECRET }),
    Credentials({
      credentials: { email: {}, password: {} },
      authorize: async (credentials) => {
        // 이메일/패스워드 검증 로직
      },
    }),
  ],
  callbacks: {
    session: async ({ session, user }) => {
      session.user.id = user.id;
      session.user.role = user.role;
      session.user.activeOrganizationId = user.activeOrganizationId;
      return session;
    },
  },
  pages: {
    signIn: "/login",
    error: "/login/error",
    verifyRequest: "/verify-email",
  },
});
```

#### 1.3.2 토큰 갱신 전략

```
[사용자 요청]
    │
    ▼
[미들웨어: 세션 검증]
    │
    ├── 세션 유효 + 만료까지 50% 이상 남음
    │   → 정상 처리
    │
    ├── 세션 유효 + 만료까지 50% 미만 (슬라이딩 윈도우)
    │   → 세션 갱신 (updatedAt 업데이트)
    │   → 세션 쿠키 maxAge 재설정
    │   → 정상 처리
    │
    ├── 세션 만료
    │   → 401 Unauthorized
    │   → 클라이언트: 로그인 페이지로 리다이렉트
    │   → "세션이 만료되었습니다. 다시 로그인해주세요."
    │
    └── 세션 무효 (DB에 존재하지 않음)
        → 401 Unauthorized + 쿠키 삭제
        → 로그인 페이지로 리다이렉트
```

**슬라이딩 윈도우 세션 갱신:**

- 세션 만료 시간의 50% 시점에서 자동 갱신
- 예: 24시간 세션 → 12시간 경과 후 첫 요청 시 갱신
- 갱신 시 `sessions.expires` 값만 업데이트 (경량 쿼리)

#### 1.3.3 보안 고려사항

**CSRF 방어:**

| 항목 | 구현 |
|------|------|
| CSRF 토큰 | NextAuth.js 내장 CSRF 토큰 사용 |
| SameSite 쿠키 | `SameSite=Lax` (기본) |
| Origin 검증 | API Route에서 `Origin` / `Referer` 헤더 검증 |
| State 파라미터 | OAuth 플로우에서 state 파라미터 검증 (NextAuth 자동 처리) |

**XSS 방어:**

| 항목 | 구현 |
|------|------|
| 세션 쿠키 | `HttpOnly`, `Secure`, `SameSite=Lax` |
| CSP 헤더 | `Content-Security-Policy` 설정 |
| 입력 검증 | Zod로 모든 사용자 입력 서버 사이드 검증 |
| 출력 이스케이프 | React의 기본 XSS 방어 + DOMPurify (에디터) |
| API 응답 | `X-Content-Type-Options: nosniff` |

**추가 보안 조치:**

| 항목 | 구현 |
|------|------|
| 비밀번호 해싱 | bcrypt (salt rounds: 12) |
| Rate Limiting | Upstash Redis 기반 sliding window |
| 보안 헤더 | Helmet.js 또는 Next.js `headers()` 설정 |
| SQL Injection | Prisma ORM 파라미터 바인딩 (기본 방어) |
| HTTPS 강제 | Vercel 기본 설정 + HSTS 헤더 |
| 민감 데이터 로깅 금지 | 패스워드, 토큰 등 로그 마스킹 |

**Next.js 미들웨어 인증 검사:**

```typescript
// src/middleware.ts
import { auth } from "@/lib/auth/config";
import { NextResponse } from "next/server";

const publicPaths = ["/", "/login", "/signup", "/verify-email", "/reset-password", "/share/"];
const adminPaths = ["/admin"];

export default auth((req) => {
  const { pathname } = req.nextUrl;
  const session = req.auth;

  // 공개 경로: 인증 불필요
  if (publicPaths.some((path) => pathname.startsWith(path))) {
    return NextResponse.next();
  }

  // 미인증: 로그인으로 리다이렉트
  if (!session) {
    return NextResponse.redirect(new URL(`/login?callbackUrl=${pathname}`, req.url));
  }

  // 관리자 경로: SUPER_ADMIN 역할 필요
  if (adminPaths.some((path) => pathname.startsWith(path))) {
    if (session.user.role !== "SUPER_ADMIN") {
      return NextResponse.redirect(new URL("/", req.url));
    }
  }

  // 정지된 사용자: 차단
  if (session.user.status === "SUSPENDED") {
    return NextResponse.redirect(new URL("/account-suspended", req.url));
  }

  return NextResponse.next();
});

export const config = {
  matcher: ["/((?!api/auth|_next/static|_next/image|favicon.ico).*)"],
};
```

- Owner는 1명만 가능, 이전 시 대상자를 Owner로 승격 후 본인은 Admin으로 강등
- Admin은 자기 이하 역할만 변경 가능 (Admin → Editor/Viewer)
- Owner만 Admin을 지정/해제 가능
- 마지막 Owner는 역할 변경 불가 (소유권 이전 필수)

### 2.2 팀 기능

#### 2.2.1 팀 PRD 공유

- 팀 워크스페이스에서 생성한 PRD는 팀 멤버 전원에게 자동 공유
- 공유 범위는 해당 워크스페이스 멤버로 한정
- PRD 목록에 작성자 표시 (아바타 + 이름)

**PRD 상태와 팀 가시성:**

| PRD 상태 | 가시성 |
|---------|--------|
| DRAFT (초안) | 작성자 + Admin/Owner만 열람 |
| IN_REVIEW (검토 중) | 팀 전원 열람 |
| FINALIZED (확정) | 팀 전원 열람 |
| ARCHIVED (보관) | Admin/Owner만 목록에 표시 |

**권한 검사 로직 (서버 사이드):**

```typescript
// src/lib/auth/permissions.ts

type Permission =
  | "prd:create"
  | "prd:read"
  | "prd:update"
  | "prd:delete"
  | "prd:delete_others"
  | "member:invite"
  | "member:remove"
  | "member:update_role"
  | "org:update"
  | "org:delete"
  | "template:manage"
  | "log:read";

const ROLE_PERMISSIONS: Record<OrgRole, Permission[]> = {
  OWNER: [
    "prd:create", "prd:read", "prd:update", "prd:delete", "prd:delete_others",
    "member:invite", "member:remove", "member:update_role",
    "org:update", "org:delete", "template:manage", "log:read",
  ],
  ADMIN: [
    "prd:create", "prd:read", "prd:update", "prd:delete", "prd:delete_others",
    "member:invite", "member:remove", "member:update_role",
    "org:update", "template:manage", "log:read",
  ],
  EDITOR: [
    "prd:create", "prd:read", "prd:update", "prd:delete",
    "log:read",
  ],
  VIEWER: [
    "prd:read",
  ],
};

export async function checkPermission(
  userId: string,
  organizationId: string | null,
  permission: Permission
): Promise<boolean> {
  // 개인 워크스페이스: 본인 데이터만 접근
  if (!organizationId) return true;

  const member = await prisma.organizationMember.findUnique({
    where: { userId_organizationId: { userId, organizationId } },
  });

  if (!member) return false;
  return ROLE_PERMISSIONS[member.role].includes(permission);
}
```

```typescript
// src/hooks/usePWAInstall.ts
export function usePWAInstall() {
  const [deferredPrompt, setDeferredPrompt] = useState<BeforeInstallPromptEvent | null>(null);
  const [showBanner, setShowBanner] = useState(false);

  useEffect(() => {
    const visitCount = parseInt(localStorage.getItem("visitCount") || "0") + 1;
    localStorage.setItem("visitCount", String(visitCount));

    const dismissed = localStorage.getItem("pwaDismissed");
    const isStandalone = window.matchMedia("(display-mode: standalone)").matches;

    // 3회 이상 방문 + 미설치 + 미거절 시 표시
    if (visitCount >= 3 && !isStandalone && !dismissed) {
      window.addEventListener("beforeinstallprompt", (e) => {
        e.preventDefault();
        setDeferredPrompt(e as BeforeInstallPromptEvent);
        setShowBanner(true);
      });
    }
  }, []);

  const install = async () => {
    if (!deferredPrompt) return;
    deferredPrompt.prompt();
    const result = await deferredPrompt.userChoice;
    setDeferredPrompt(null);
    setShowBanner(false);
  };

  const dismiss = () => {
    setShowBanner(false);
    localStorage.setItem("pwaDismissed", Date.now().toString());
  };

  return { showBanner, install, dismiss };
}
```

- 토글 버튼으로 접기/펼치기
- 접은 상태에서 메인 영역이 전체 너비 사용
- 접은 상태에서도 "가이드 열기" 버튼 표시

**사이드바:**

- 태블릿에서는 사이드바 기본 숨김
- 햄버거 메뉴(≡) 클릭 시 오버레이로 표시
- 바텀 네비게이션은 표시하지 않음 (사이드바 오버레이 사용)

### 5.5 공통

#### 5.5.1 브레이크포인트 전환 시 상태 유지

| 상태 | 전환 시 동작 |
|------|-------------|
| 현재 Step | 유지 (URL 파라미터) |
| 입력 내용 | 유지 (Zustand 스토어) |
| 미리보기 패널 토글 | 데스크톱↔태블릿 전환 시 기본 상태로 리셋 |
| 스크롤 위치 | 최대한 유지 (동일 Step 내) |
| 모달 상태 | 유지 |
| 사이드바 상태 | 브레이크포인트 기본값으로 리셋 |

**구현 원칙:**

- 사용자 입력 데이터는 항상 유지 (Zustand 퍼시스트)
- UI 레이아웃 상태는 CSS 미디어 쿼리로 제어 (JS 불필요)
- 네비게이션 상태는 URL 기반 (뒤로가기 지원)

  adminId         String    // 관리자 사용자 ID
  action          String    @db.VarChar(100) // "user.suspend" | "user.delete" | "user.role_change" | "template.create" | ...
  targetType      String    @db.VarChar(50) // "user" | "template" | "notice" | "organization"
  targetId        String    // 대상 ID
  details         Json?     // 변경 상세 (이전 값, 새 값 등)
  ipAddress       String?   @db.VarChar(45)

  createdAt       DateTime  @default(now())

  admin           User      @relation("AdminActor", fields: [adminId], references: [id])

  @@index([adminId, createdAt])
  @@index([targetType, targetId])
  @@index([action])
  @@index([createdAt])
  @@map("admin_audit_logs")
}

/// 공지사항
model Notice {
  id              String               @id @default(cuid())
  title           String               @db.VarChar(100)
  content         String               @db.Text // Markdown
  type            NoticeType           @default(INFO)
  displayPosition NoticeDisplayPosition @default(IN_APP)
  target          NoticeTarget         @default(ALL)
  isActive        Boolean              @default(true)

  startsAt        DateTime             @default(now())
  endsAt          DateTime?

  createdById     String               // 관리자 ID
  createdAt       DateTime             @default(now())
  updatedAt       DateTime             @updatedAt

  @@index([isActive, startsAt, endsAt])
  @@index([type])
  @@map("notices")
}

/// 피드백/문의
model Feedback {
  id              String         @id @default(cuid())
  userId          String?        // 비로그인 문의 가능
  type            FeedbackType   @default(OTHER)
  status          FeedbackStatus @default(PENDING)
  title           String         @db.VarChar(200)
  content         String         @db.Text
  email           String?        @db.VarChar(255) // 비로그인 시 연락처

  // 관리자 응답
  adminResponse   String?        @db.Text
  respondedById   String?        // 응답한 관리자 ID
  respondedAt     DateTime?

  // 메타데이터
  userAgent       String?        @db.VarChar(500)
  pageUrl         String?        @db.VarChar(500) // 문의 발생 페이지

  createdAt       DateTime       @default(now())
  updatedAt       DateTime       @updatedAt

  user            User?          @relation(fields: [userId], references: [id])

  @@index([status])
  @@index([type])
  @@index([userId])
  @@index([createdAt])
  @@map("feedbacks")
}
